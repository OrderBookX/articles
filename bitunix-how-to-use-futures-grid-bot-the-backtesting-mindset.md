# How to Use Futures Grid Bot — The Backtesting Mindset

Most grid traders configure parameters based on gut feeling. Better traders use a backtesting mindset — even without a formal backtesting tool, you can simulate how a grid would have performed using historical data.

## The Manual Backtest Method

Pull up a 30-day daily chart for your pair. Identify a 14-day consolidation period. Mark the range (high and low). Now mentally overlay a grid: if you had placed 25 levels across that range at 5x leverage, how many times would price have crossed each level? Count the approximate cycles. Multiply by estimated per-cycle profit. Subtract estimated fees.

### What This Tells You

This rough calculation reveals whether the range was active enough for a grid to be profitable. If price only crossed 5 levels in 14 days, a 25-grid setup would have been mostly idle. If price crossed 15+ levels regularly, the grid would have been highly active.

## Applying the Backtest to Live Setup

The range that performed best historically becomes your starting point for a live grid. Adjust for current conditions — if volatility has increased since your backtest period, widen the range slightly. If it's decreased, tighten it.

### Key Insight

The backtest doesn't predict the future. But it tells you whether your parameters are reasonable for the type of price action your pair typically exhibits. A grid configured for a range that never existed historically has zero edge.

## Forward Testing on Small Capital

After your mental backtest suggests viable parameters, deploy a small-capital grid as a forward test. Use $200 to $500. Run it for one to two weeks. Compare actual performance against your backtest estimate. The gap between expected and actual reveals what your analysis missed.
